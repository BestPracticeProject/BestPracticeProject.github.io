О стандартах

Данные стандарты относятся к компании Workle и предназначены для разработчиков данной компании.

Данные стандарты требуют обязательный пересмотр каждые 6 месяцев.

Правила дополнения / пересмотрa стандартов
Для того, чтобы внести дополнения/изменения в документ со стандартами, необходимо:
  1. Создать pull request. В поле “Message” описать в свободной форме предпосылки вносимых изменений.
  2. После необходимо сообщить о дополнениях/изменениях в чате Front-End Workle и оставить ссылку на соответствующий pull request.
  3. На рассмотрение данного pull request вашим коллегам дается до 3-х дней. Если необходимо больше, то в поле “Message” (см.п.1) необходимо указать дату окончания рассмотрения.
  4. После того, как вы ответили на все комментарии и по истечению 3-х дней pull request может быть смёржен.

Введение

Руководство описывает правила для оформления и форматирования HTML/CSS/JS кода.

Цель – повысить качество кода, удобочитаемость и дальнейшую поддержку.

Данное руководство относиться к используемым файлам HTML/CSS/JS. Старайтесь строго соблюдать ниже описание правила.

В сомнительных ситуациях руководствуйтесь логикой и общепринятыми принципами.

Общее

Вёрстка должна полностью соответствовать макету в соответствии 1:1 (допускается расхождение в 5px для текста), а конечный результат техническому заданию;
Проверить соответствие: Pixel Perfect (аддон Mozilla) PerfectPixel (аддон Chrome) ModularGrid (б-н)
Корректное отображение и работа скриптов в браузерах процент использования которых больше двух (2%) процентов;
Используйте анализаторы качества кода (валидаторы);
Валидаторы: HTML W3C CSS W3C CSSLINT JSHint пояснения ошибок JSHint
У используемых шрифтов должны быть прописаны стандартные Win/Linux/Mac-аналоги и общее семейство шрифтов.
Статья
Изображения до 1кб должны быть конвертированы в Base64, а остальные оптимизированны с помощью Панды.
Base64 Панда

HTML

Тип документа и кодировка

Используйте HTML5 во всех html-документах: c типом контентом text/html и кодировку UTF-8.
w3

Форматирование

Выделяйте новую строку для каждого блочного, табличного или списочного элемента и ставьте отступы для каждого дочернего элемента, независимо от заданых стилей, а также ставьте отступ для каждого вложеного элемента. 
<div>
	<p>Hello, 
		<span>Workle</span>
	</p>
	<ul>
		<li>I am alone</li> 
	</ul>
</div>

Отступы

Не смешивайте табуляцию и пробелы.
Для отступов используйте табуляцию;
Удаляйте пробелы в конце строки.

Регистр

Весь код должен быть в нижнем регистре, кроме текста.

Закрываете одиночные теги

<link rel="stylesheet" href="/b-crosselling.css" />
<img src="/" alt=”” />
<input />
w3

Формы

У полей должен быть задан правильный порядок переключения через tabindex;
w3
Label и input/select должны быть слинкованы;
Поля, которые можно валидировать с помощью HTML5 нужно валидировать (pattern, email и т.д);
Дополнительно поля обладают валидацией с помощью JS.

Элементы

Используйте элементы семантически осмыслено (h1,h2..h6, p и т.д);
Не указывайте атрибут type при подключении CSS и JS в документ;
text/css text/javascript
Всегда указывайте alt у медиа. Если текст избыточен или картинка используется в декоративных целях, то указывайте пустой альтернативный текст (alt);
w3
У всех пиктограмм и элементов интерфейса, значение которых может быть не до конца очевидно указывайте title с подсказкой;
У внешних ссылок всегда указывайте target="_blank";

CSS

Группируйте правила по смыслу или используйте CSSComb
CSScomb
При написании правила для нескольких селекторов помещайте каждый селектор на новой строке; 
.b-super__man,
.b-super__woman,
.b-super__child {
...
Перед открывающей скобкой правила ставьте один пробел;
Внутри блока каждое правило помещайте на новой строке, а так же ставьте точку с запятой после последнего правила в блоке; 
.b-batman {
	display: block;
	background: #000 url("logo/logo_on_the_sky.png") no-repeat;
	color: #ffff00;
}
Используйте сокращенные формы записи свойств, где это возможно; 
.b-spiderman {
	font: bold 18px/1 “Spider_font”, georgia, serif;
}
Опускайте единицу измерения при значении 0 у свойства, а так же не ставьте "0" в целой части дробных чисел; 
.b-captain_america {
	margin: 0;
	padding: 0 .8px;
}
Ставьте закрывающую скобку на одной вертикальной линии с первым символом в селекторе;
Используйте двойные кавычки;
Используйте шестнадцатеричные значения в нижнем регистре, кроме случая использования box-shadow - для него применяеться rgba();
Модификатор блока пишите с отступом в Tab относительно блока;
Используйте только классы (".something"), а идентификаторы ("#something") используйте для создания якорей.
